```bash
npm run lint         # ESLint
```


## Architecture

The codebase enforces a strict **no-upward-import rule**: `core/` has zero UI or React dependencies; `features/` and `ui/` can import from `core/` but not vice versa.

```
src/
├── core/        # Pure TypeScript — geometry engines, math, domain types, utils
├── features/    # React features, each owning: components/, hooks/, logic/, types/
├── ui/          # Rendering layers and reusable UI components
├── io/          # Data parsing (graph6 catalogs, ribbon-knot .txt files)
└── app/         # React entry point, initial scene setup
```

### Core geometry modules

- **`core/geometry/envelope/`** — `EnvelopeComputer` interface + `ElasticEnvelope` (elastic band) + `contactGraph.ts` (outer tangents between disks)
- **`core/geometry/dubins/`** — Bitangent path computation with asymmetric disk radii
- **`core/geometry/hull/`** — `robustHull.ts` and `sequenceHull.ts` (topology-aware convex hull)
- **`core/geometry/rolling/`** — Rolling disk simulation geometry
- **`core/types/`** — Canonical domain types: `entities.ts` (Point, Segment, Arc, Scene), `knot.ts` (Crossing, Region, ContactGraph), `cs.ts` (CSDisk, CSDiagram)

### Feature modules

Each feature module is self-contained. Key ones:

- **`features/editor/`** — Main orchestrator; `useEditorState` is the central state hook; `EditorPage.tsx` wires everything
- **`features/knot/logic/KnotEnvelopeComputer.ts`** — Implements `EnvelopeComputer` with a 4-tier fallback: elastic band → relaxed elastic → sequence hull → robust hull
- **`features/ribbon/`** — Ribbon-knot mode (Ayala-Kirszenblat-Rubinstein model); data loaded via `io/loadRibbonKnots.ts`
- **`features/gallery/`** and **`features/catalog/`** — Gallery of saved configurations; graph6 catalog browser

### Rendering

SVG-based. Rendering is split into mode-specific layers in `ui/renderer/layers/`:
`BaseLayer` → `StandardLayer`, `KnotLayer`, `DubinsLayer`, `RollingLayer`, `RibbonLayer`, `PathLayer`

### Data I/O

- Graph catalogs (graph6 format): `public/data/graphs*.txt`, parsed by `io/parseGraph6.ts`
- Ribbon-knot files (.txt JSON): generated by `gen_knots.js`, loaded by `io/loadRibbonKnots.ts`

### Path alias

`@/` resolves to `src/` (configured in `vite.config.ts` and `tsconfig.json`).
